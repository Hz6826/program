zb小程序

欢迎来到zb小程序插件开发文档，当前为测试阶段，可能有较大变动，请注意！
插件是zb小程序3.3.0版本后为开发者提供的程序接口，插件列表位于存储库/addon/addon.json目录，程序内可安装、删除、更新插件，插件安装目录为Windows系统的User/zb/addon文件夹

插件格式

插件为文件夹形式，文件夹名字为插件ID，插件ID可以由大小写字母、数字及下划线，尽量不要使用其他字符。
文件夹下有addon.json文件，其中包括插件相关信息，实例如下：
{
"id": "appstore", // 插件ID要求同上
"name": "应用商店", // 插件名称，建议简洁明了
"version": "1.0.1", // 版本号，注意尽量是三段式，不要有除数字和.的符号
"file": [ // 为列表形式，储存着项目所有文件的相对（addon.json的）路径（或绝对网址），必需有__init__.py
"init.py" // 文件格式不限，如果为zip文件会自动解压到程序的addon文件夹（由于Github Pages无法下载双下划线开头的文件，请将__init__.py改为init.py，其他方式网站无需此操作）
],
"lib": [] // Python第三方运行库列表，会在插件安装时自动安装
}

Python格式

__init__.py文件开头需要添加以下代码来调用主程序API：
import sys, os
sys.path = [os.path.dirname(sys.argv[0])] + sys.path
from source.custom import *
os.chdir(os.path.dirname(__file__))
插件主页必须设置为class AddonTab():，同时还需要设置self.title和self.subtitle作为页面标题和子标题
self.setIcon()可设置页面图标，可为FIF.图标或本地图片文件
self.setObjectName("")可设置页面名称

发布方式

插件可选择将整个文件夹公开上传到任意无反爬虫网站，使得每个文件的访问网址与addon.json文件夹中“file”列表的对应关系一致。
也可以选择将整个文件夹压缩为无密码zip文件（注意压缩的是整个文件夹而不是全选里面的文件），与addon.json同时上传至服务器，zip文件的访问网址与addon.json文件夹中“file”列表的对应关系一致。
发布后在Github上面向作者申请并提供addon.json网址，作者会添加插件至索引中，请确保下载链接长时间稳定有效！
